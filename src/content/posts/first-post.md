在 Web 开发领域，我们似乎陷入了一种悖论：为了让网站更快，我们不断地加入更多的 JavaScript——框架、库、打包器、优化器，层层叠叠。[^1] 最终，用户下载的代码量反而呈指数级增长。

[^1]: 这一现象被业内称为「工具膨胀」（Tooling Bloat），具体数据可参考 HTTP Archive 的年度状态报告。

## 回归 HTML 的本质

超文本标记语言（HTML）被发明出来的初衷，就是作为一种**文档格式**。它不应当是一台虚拟机的字节码，而应当是可读、可索引、可直接渲染的语义结构。

Astro 的核心洞察在于：**大多数内容不需要交互性**。一篇博客文章、一张项目截图、一段作者简介——这些元素天生就是静态的。我们为什么要为它们支付 React 或 Vue 的运行时成本？

## 排版即设计

当技术栈被简化到极致，设计的重心自然回归到**文字本身**。Tailwind 的 `prose` 插件提供了一套优雅的默认排版，但真正的精品博客需要更进一步的定制。

> "好的排版是透明的。读者应当沉浸在内容中，而不会意识到字体、行高和页边距的存在。"

中文排版的特殊性在于：

1. **行高需要更大**：1.75 是底线，1.8 更佳
2. **避免 Italic**：中文斜体观感极差，应使用字重变化代替
3. **段落缩进或段间距**：现代网页多采用段间距，保持呼吸感

## 暗色模式的色彩科学

暗色模式不是简单地把背景变成 `#000`。对于 OLED 屏幕，纯黑确实省电；但对于 LCD 屏幕，纯黑反而会因为高对比度造成视觉疲劳。

我们采用的 `#121416` 带有极微弱的蓝灰倾向，它模拟了**优质铜版纸在暗房中的反光质感**，而非电子屏幕的自发光。

## 结语

构建这个博客的过程，本质上是一次**减法工程**。我们删除了 React、删除了状态管理、删除了 hydration、删除了运行时。剩下的，只有内容本身——以及承载内容的最小必要结构。

这，才是 Web 应有的样子。
